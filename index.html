Legacy Code

Legacy Code - Defined

"Legacy Code is often used as a slang term for difficult-to-change code that we don't understand. To me Legacy Code is simply code without tests. Without tests we don't know if our changes are making things better or worse."

Michael Feathers, Working Effectively with Legacy Code, 2005

Case Study : Travel 2

The Problem

Enhance bespoke Legacy Airline & Hotel Reservation System

-v-

Purchase Off-The-Shelf Solution

Rationale for change

  1. Staff Training Costs
  2. Greenscreen Interface
  3. High Bus Factor
  4. Creaking Infrastructure - parts from Ebay!!!
  5. Difficult to change and react
  6. Vendor golddust

Existing

Architecture

My Recommendations

  1. Port infrastructure to new servers
  2. Create XML-RPC interface into the Business Layer
  3. Create Modern Web Interface
  4. Slowly document, understand the BL layer

Reasoning

  1. It's a mistake to replace your business differentiator with an off-the-shelf solution
  2. Don't choose an off-the-shelf solution if >20% of the project is customization
  3. Cheaper and Safer
  4. No big bang, incremental roll out. Hotels first.

Rules of Thumb

  1. YAGNI - If in doubt choose the simplest solution
  2. MVP - Create the Minimum Viable Product as quickly as you can
  3. Play to your team strengths
  4. -1000 points rule - Anders Hejlsberg
  5. Be aware of the realpolitik
  6. Confirmation Bias - Impossible to eliminate

Conclusions

  3. Hindsight is a wonderful thing
  4. Get on the plane - Seth Godin
  5. Doing nothing is not an option
